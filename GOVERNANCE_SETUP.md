# AIGuardian 社内導入ガイド（情報システム部門向け）

**AIGuardian** は、社内に乱立したAIエージェント・カスタムGPTのガバナンスを一元化する
コントロールダッシュボードです。2つのエンジンで構成されます。

| エンジン | 役割 |
|---|---|
| **ShadowAI_Detector** | エージェント台帳のプロンプト・参照ソースをベクトル化スキャンし、機能重複・野良AI・古い規程依存・危険指示を検知 |
| **LoopBreaker_Proxy** | エージェント間通信を常時監視し、同一パターンのラリーを3回検知した瞬間に通信を強制遮断・管理者へ通知 |

---

## 1. 動作要件

- Python 3.10 以上（3.13 で動作確認済み）
- 依存パッケージは `fastapi` / `uvicorn` のみ（**閉域網でも導入可能**。MLライブラリ・外部API不要）
- メモリ 512MB / 1vCPU 程度の小規模サーバで十分

## 2. インストールと起動

```bash
# 1. 配布物を社内サーバに配置
cd aiguardian/

# 2. 依存パッケージのインストール
pip install -r requirements.txt

# 3. 起動（ブラウザが自動で開きます）
python app.py
```

→ `http://127.0.0.1:8787` にダッシュボードが立ち上がります。

### 起動オプション（環境変数）

| 環境変数 | 既定値 | 説明 |
|---|---|---|
| `AIGUARDIAN_PORT` | `8787` | リッスンポート |
| `AIGUARDIAN_WEBHOOK_URL` | （なし） | ループ遮断時に通知をPOSTするWebhook URL（Slack Incoming Webhook等） |
| `AIGUARDIAN_NO_BROWSER` | `0` | `1` でブラウザ自動起動を抑止（サーバ常駐運用時） |

```bash
# 例: Slack通知付きでポート9000で常駐起動
AIGUARDIAN_PORT=9000 \
AIGUARDIAN_WEBHOOK_URL=https://hooks.slack.com/services/XXX/YYY/ZZZ \
AIGUARDIAN_NO_BROWSER=1 \
python app.py
```

## 3. 動作確認（5分でできる受け入れテスト）

1. ダッシュボードを開き、KPIカードに「登録エージェント 8」と表示されることを確認
2. **「⚡ 無限ループをシミュレート」** ボタンを押す
   - 通信フィードに自動返信のラリーが流れ始める
   - **同一パターン3回目のラリーで「🚫 遮断」** に切り替わり、画面右下にトースト通知が出る
   - 「管理者通知ログ」に遮断通知が記録される（Webhook設定時はSlackにも届く）
   - 以降の送信試行は「拒否」となり、APIコストが発生しないことを確認
3. **「✉️ 正常トラフィックをシミュレート」** を押し、健全な会話は遮断**されない**ことを確認
4. 遮断カードの「✅ 確認済み — 遮断を解除」で通信が再開できることを確認
5. ロジックの回帰テストはいつでも実行可能:

   ```bash
   python tests/test_aiguardian.py   # 17件のセキュリティテスト
   ```

## 4. 実データの接続方法

### 4.1 エージェント台帳（ShadowAI_Detector）

`data/agents.json` を配置すると、サンプルデータの代わりに読み込まれます
（配置後はダッシュボードの「🔍 台帳を再スキャン」を押下）。

```json
[
  {
    "id": "agt-101",
    "name": "経費精算ボット",
    "owner": "山田 太郎",
    "department": "経理部",
    "model": "claude-sonnet-4-6",
    "monthly_cost_jpy": 12000,
    "created_at": "2026-01-10",
    "prompt": "（エージェントのシステムプロンプト全文）",
    "sources": [
      { "name": "経費精算規程 2026年版", "last_updated": "2026-04-01" }
    ]
  }
]
```

**台帳の集め方の例:**
- 社内のカスタムGPT管理画面 / Difyなどのエージェント基盤からエクスポート
- API Gatewayのアクセスログからエージェント名・コストを集計してマージ
- `owner` / `department` が埋められないものは、そのまま **「野良AIの疑い」として検知される**のが正しい挙動です

### 4.2 エージェント間通信（LoopBreaker_Proxy）

社内エージェントの送信処理を、直接送信ではなく本プロキシ経由に変更します。

```
POST http://<サーバ>:8787/api/proxy/message
Content-Type: application/json

{ "sender": "エージェントA", "recipient": "エージェントB", "content": "本文" }
```

レスポンスの `status` で送信可否を判定してください。

| status | 意味 | エージェント側の処理 |
|---|---|---|
| `forwarded` | 通過 | そのまま送信してよい |
| `blocked` | **今このメッセージでループ確定 → 遮断発動** | 送信を中止し、処理を停止 |
| `rejected` | 遮断中ペアへの送信試行 | 送信を中止（管理者の解除待ち） |

**組み込みパターン例:**
- メール自動返信Bot: SMTP送信の直前にプロキシへ問い合わせ、`forwarded` 以外なら送信しない
- チャットBot（Slack/Teams）: 投稿APIをラップした共通SDKにプロキシ呼び出しを挟む
- LangChain等のエージェントフレームワーク: ツール実行のコールバックでフックする

## 5. 検知ロジックとチューニング

### ShadowAI_Detector（`aiguardian/detector.py`）

| 定数 | 既定値 | 意味 |
|---|---|---|
| `DUPLICATE_THRESHOLD` | `0.60` | この類似度以上のプロンプトペアを重複と判定（TF-IDF + コサイン類似度） |
| `STALE_SOURCE_DAYS` | `365` | 参照ソースがこの日数を超えて未更新なら鮮度切れ |
| `DANGEROUS_PATTERNS` | — | ジェイルブレイク指示・APIキー埋め込み・認証回避・機密外部送信などの検知ルール（自社ポリシーに合わせて追記可能） |
| `DEPRECATED_PATTERNS` | — | 改正前規程・廃止済みモデルAPIなどの「古い根拠」検知ルール |

リスクスコアは加点式（危険指示+40、廃止規程+25、野良AI+25、鮮度切れ+15、重複+10）で、
**60点以上=重大 / 30点以上=要注意** としてダッシュボードに色分け表示されます。

### LoopBreaker_Proxy（`aiguardian/loopbreaker.py`）

| 定数 | 既定値 | 意味 |
|---|---|---|
| `REPEAT_THRESHOLD` | `3` | 同一パターンがこの回数現れたら遮断 |
| `SIMILARITY_THRESHOLD` | `0.72` | 「同一パターン」とみなす本文類似度（Jaccard）。日付・整理番号などの数値揺れは正規化で吸収済み |
| `HISTORY_WINDOW` | `12` | ペアごとに監視する直近メッセージ数 |
| `COST_PER_1K_TOKENS_JPY` | `0.45` | コスト推計の単価。自社の契約単価に合わせて変更 |

**誤検知が出る場合**は `SIMILARITY_THRESHOLD` を上げる（例 0.85）、
**検知漏れが心配な場合**は下げる（例 0.6）方向で調整してください。

## 6. 推奨運用フロー

1. **週次**: ダッシュボードの「機能重複クラスタ」を確認し、ムダ費用の大きいクラスタから統合を所有者に打診
2. **即時**: 重大アラート（危険指示・野良AI）は所有者特定 → 利用停止 or プロンプト修正を依頼
3. **遮断発生時**: 管理者通知を受けたら遮断カードの内容（ラリー内容・消費コスト）を確認し、
   エージェント側の自動返信条件を修正してから「遮断を解除」
4. **四半期**: `DEPRECATED_PATTERNS` に廃止した社内規程・システム名を追記し、再スキャン

## 7. セキュリティ上の注意

- 既定では `127.0.0.1` のみにバインドします。部門公開する場合はリバースプロキシ（nginx等）配下で
  **社内認証（SSO/Basic認証）を必ず挟んでください**。ダッシュボード自体に認証機能はありません
- `data/agents.json` にはプロンプト全文（機密になり得る）が含まれます。サーバのアクセス権を限定してください
- 状態はインメモリ管理です。プロセス再起動で遮断状態・イベントはリセットされます
  （遮断の永続化が必要な場合はDB接続の拡張を検討）

```
├── aiguardian/

│   ├── loopbreaker.py         # LoopBreaker_Proxy（ループ遮断エンジン）
│   └── sample_data.py         # デモ用台帳・シミュレーションシナリオ

├── tests/test_aiguardian.py   # セキュリティロジック回帰テスト（17件）
└── data/agents.json           # （任意）実台帳を配置するとサンプルより優先
```
